# Coding with AI: a hands-on workshop for researchers

Python and R. Browser-based. Designed for researchers who are new to AI-assisted coding.

The canonical live format is one hour. [SCRIPT_1HR.md](SCRIPT_1HR.md) supplies the core route;
[ADVANCED_PATHS.md](ADVANCED_PATHS.md) supplies alternatives for experienced participants.
Participants practice the following:

- an NLP pipeline that turns picture-description transcripts into quantitative measures (Python)
- a statistical model of aphasia severity, and a publication-ready figure (R)
- a working framework for identifying when an AI assistant's output is unreliable

The dataset is synthetic and generated by `data/make_data.py`. It contains no participant data,
but the workshop's synthetic-data status does not determine what is permissible for a real study.
Institutional policy, the data-use agreement, and the approved service govern that decision.

## The one idea

The workshop treats AI output as a draft that needs an analytic check. Participants specify the
data structure and design, run the code, inspect the resulting objects, and trace written claims
to the output that supports them. The workshop does not evaluate a particular AI system.

## Setting this up as the facilitator

1. Use the repository under the GitHub account named in `python/_build_notebooks.py` and
   `python/_build_advanced_notebook.py`. Rebuild the notebooks after changing that account name.
2. In Posit Cloud, create a project from the GitHub repository. Run `r/install_packages.R`, then
   test `r/02_r_solution.R`.
3. Share the Posit project with view access. Participants make a permanent copy before editing.
4. Test the Colab link, the advanced Colab link, and the Posit copy flow in a private browser
   session after the final GitHub push. Freeze the repository before participants make copies.

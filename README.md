# Coding with AI: a hands-on workshop for researchers

**Python + R. No prior programming experience required. Nothing to install.**

The full curriculum runs for three hours. [SCRIPT_1HR.md](SCRIPT_1HR.md) provides the
one-hour core route, with optional fast-track branches for experienced participants.
Across the full curriculum, participants build the following with AI assistance:

- an NLP pipeline that turns picture-description transcripts into quantitative measures (Python)
- a statistical model of aphasia severity, and a publication-ready figure (R)
- a working framework for identifying when an AI assistant's output is unreliable

The dataset is **synthetic**, generated by `data/make_data.py`; it is not real patient
data. This allows unrestricted use in chatbots and serves as the basis for the
workshop's data-privacy discussion.

## The one idea

An AI assistant produces fast, confident output that is sometimes wrong. It has
been trained on a large volume of text but has no understanding of your specific
data or design.

It will save time on routine coding tasks. It will also, without any signal of
uncertainty, drop controls into a model where they don't belong, run a t-test on
repeated measures, or write a Results paragraph containing a fabricated p-value.

The core competency this workshop teaches is not prompting but verification: checking
AI output against the data, the model object, and the design before trusting it.

## Setting this up as the facilitator

1. Create a public GitHub repo called `ai-coding-workshop` and upload these files
   (github.com → New repository → "uploading an existing file" → drag the folder in.
   No git required.)
2. Find-and-replace `mjmarte` with your GitHub username in `README.md` and in
   `python/_build_notebooks.py`, then re-run that script to rebuild the notebooks
   with the right clone URL.
3. On [posit.cloud](https://posit.cloud): New Project → **New Project from Git
   Repository** → paste your repo URL. Open it, run `r/install_packages.R` once, then
   share the project (Access → everyone, "view" permission). Attendees click your link
   and hit **Save a Permanent Copy** — they get the packages already installed.
4. Test both links from a private browser window before the day.
